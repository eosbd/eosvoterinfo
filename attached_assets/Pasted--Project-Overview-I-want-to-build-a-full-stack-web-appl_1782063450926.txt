​Project Overview:
I want to build a full-stack web application to host and manage Bangladesh Voter Information. The system will have a public-facing search interface, a secure Admin Panel with a dashboard, an automated data extraction/upload system, and a public API for third-party integrations.
​Tech Stack Recommendation:
​Backend: Python (FastAPI or Flask) - Best for handling OCR, file processing, and text encoding.
​Database: SQLite (for development) or PostgreSQL.
​Frontend: HTML, CSS (TailwindCSS), and Vanilla JS or React.js.
​Data Extraction: pdfplumber, pandas, or pytesseract (with Bengali language support).
​1. Core Features & Routes:
​Search Interface (Public):
​Basic Search 1: Search by exact "Voter Number" (ভোটার নং) to display the specific person's full details.
​Advanced Search: Search by combining multiple fields like "Name, Father's Name, District, and Thana" to fetch matching records.
​API Endpoint:
​Create a REST API endpoint (e.g., /api/v1/search-voter) that accepts GET/POST requests with search parameters and returns JSON data.
​2. Admin Panel (Protected Route):
​Authentication: Secure login system (Username & Password) for the admin.
​Dashboard: A visual dashboard displaying:
​Total number of voters.
​Voters grouped by District or Ward (Bar/Pie Chart).
​File Upload & Management:
​Admin can upload files in Zip, PDF, Excel (.xlsx), and Word (.docx) formats.
​A table interface to view, edit, or delete existing voter records from the database.
​3. Special Data Extraction & Text Encoding Handling (CRITICAL):
​The uploaded documents contain Bengali text.
​Encoding Fix Requirement: Raw files might have corrupted ANSI/Bijoy encodings or broken Unicode (e.g., the text might look like "Ïজলা" instead of "জেলা", or "নারায়ণগý" instead of "নারায়ণগঞ্জ"). The backend parsing logic MUST handle these anomalies, sanitize the text, convert it to standard Unicode (UTF-8), and store it perfectly in the database without any "???" or broken characters.
​The extraction logic must map the extracted text to a highly detailed database schema.
​4. UI/UX & Typography Requirements:
​The design should be clean, responsive, and modern.
​Fonts: Load "SolaimanLipi" for Bengali text and "Noto Sans" for English text.
​Data results must be displayed in a clean, categorized profile card format (Personal Details, Address Details, and Area Details).
